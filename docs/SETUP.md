# 隨時資料庫 — 部署設定指南

跟著這份文件做,大概 20~30 分鐘可以全部弄好。分四部分:
**A. Apps Script 後端**(必做) → **B. PWA 拍照 App**(必做) → **C. 日常使用**(部署完怎麼用)→ **D. 維護工具**(選用)。
錄音的部分是另一份文件,見文末連結。

---

## A. 部署 Apps Script 後端

### A-1. 建立專案

1. 打開 [script.google.com](https://script.google.com/) → 「新專案」
2. 專案名稱改成「隨時資料庫」(左上角,點一下標題就能改)

### A-2. 貼上程式碼

1. 左側檔案列表看到 `Code.gs`,把裡面內容全部刪掉,貼上 [apps-script/Code.gs](../apps-script/Code.gs) 的完整內容
2. 左側點齒輪圖示「專案設定」,勾選「在編輯器中顯示「appsscript.json」資訊清單檔案」
3. 回到編輯器,會多出 `appsscript.json` 檔案,把內容換成 [apps-script/appsscript.json](../apps-script/appsscript.json) 的內容
4. 上方「儲存專案」(Ctrl+S)

### A-3. 執行初次設定

1. 上方函式下拉選單選 `setup`,按「執行」
2. 第一次執行會跳出 Google 授權畫面:「這個應用程式未經 Google 驗證」→「進階」→「前往 隨時資料庫(不安全)」→ 全部允許
   - 這是正常的,因為是你自己的程式,不是第三方 App
3. 執行完成後,下方「執行記錄」會印出重要資訊,**把這幾行存到你的密碼管理工具或筆記(下一步 PWA 設定會用到)**:
   - 根資料夾網址
   - 索引表網址
   - **收件夾 ID**(iOS 捷徑會用到)
   - **PASS_KEY**(PWA 和捷徑都要填這組)

### A-4. 安裝自動掃描排程

1. 函式下拉選單改選 `installTriggers`,按「執行」
2. 執行記錄顯示「已安裝排程」即完成 — 之後收件夾裡的錄音檔會每 5 分鐘自動被歸檔一次

### A-4b. 安裝每日自動對帳排程

1. 函式下拉選單改選 `installCleanupTrigger`,按「執行」
2. 執行記錄顯示「已安裝排程」即完成 — 之後每天凌晨 3 點會自動**對帳**一次:補登你直接放進 Drive 的檔案、清掉已刪除(含垃圾桶)的索引列、同步課程清單、核銷「待清理提醒」。等於就算你什麼都不做,系統每天也會自己把索引表跟 Drive 對齊一次。
   - 想立刻對帳、不想等到凌晨的話,只要打開 App(或從其他 App 切回來),它就會在背景自動同步一次(詳見 C 日常使用)。

### A-5. 設定課程清單

1. 函式下拉選單改選 `setCourses`,**不要直接執行**,因為要先改參數
2. 改用「編輯器」直接在最下面暫時加一行測試呼叫,或是更簡單:上方選單「檢視 > 執行記錄」旁邊有個「執行」按鈕旁的下拉,選「setCourses」後點執行前,無法直接傳參數 — 最簡單做法是暫時修改 `setup()` 旁邊新增一個一次性函式:

   在 Code.gs 最下面暫時加入:
   ```js
   function _initCourses() {
     setCourses(['未分類', 'RAG課', 'Agent課']); // 改成你自己的課程名稱
   }
   ```
   存檔後執行 `_initCourses`,之後這個暫時函式可以刪掉不影響系統。
3. 之後想增減課程,同樣的方式改清單重跑就好(不用重新部署)

### A-6. 部署成網頁應用程式

1. 右上角「部署」→「新增部署作業」
2. 齒輪圖示選「網頁應用程式」
3. 說明填「v1」
4. **執行身分**:我(你的帳號)
5. **誰可以存取**:任何人 — 這一步務必選「任何人」,不然手機端呼叫不到
6. 「部署」→ 會再跳一次授權畫面,照 A-3 的方式允許
7. 完成後複製「網頁應用程式」網址,格式類似:
   `https://script.google.com/macros/s/AKfycb.../exec`
   **這就是 GAS_URL,PWA 設定要填這個**

   > 之後如果修改 Code.gs 想更新,要「管理部署作業」→ 編輯 → 版本選「新版本」→ 部署,網址會維持不變。

## B. 部署 PWA 拍照 App

PWA 就是 `pwa/` 資料夾,用 GitHub Pages 免費架設(跟你的屁孩特攻隊專案同一套做法)。

### B-1. 建 GitHub repo 並上傳

```bash
cd 隨時資料庫
git init
git add pwa docs apps-script
git commit -m "隨時資料庫初版"
git branch -M main
git remote add origin https://github.com/<你的帳號>/suishi-database.git
git push -u origin main
```

（沒有 repo 的話先到 GitHub 網站建一個新的 public repo，名字例如 `suishi-database`。）

### B-2. 開啟 GitHub Pages

GitHub Pages 的「Deploy from a branch」不支援選任意子資料夾(只能選 root 或 `/docs`),但我們的 PWA 檔案放在 `pwa/` 子資料夾,所以改用 repo 裡已經寫好的 **GitHub Actions 流程**([.github/workflows/deploy-pwa.yml](../.github/workflows/deploy-pwa.yml))自動部署,push 到 `main` 就會自動把 `pwa/` 部署上去,不用每次手動操作。

1. repo 頁面 → Settings → Pages
2. Source 選「**GitHub Actions**」(不是「Deploy from a branch」)
3. 存回 B-1 的 push 完成後,repo 頁面 → Actions 分頁應該會看到「Deploy PWA to GitHub Pages」這個工作流程自動跑起來,跑完後回 Settings → Pages 就會出現網址,例如:
   `https://<你的帳號>.github.io/suishi-database/`
4. 之後只要 `pwa/` 資料夾有變動並 push 到 `main`,就會自動重新部署,不用再手動按什麼

### B-3. 手機上設定並加到主畫面

1. iPhone 用 Safari 打開上面那個網址
2. 點畫面最下面「連線設定」
3. 填入:
   - **Apps Script 網址**:A-6 拿到的 GAS_URL(結尾是 `/exec`)
   - **通行金鑰**:A-3 拿到的 PASS_KEY
4. 儲存後應該會看到課程按鈕出現(A-5 設定的清單)
5. 分享圖示(方框+箭頭)→「加入主畫面」→ 命名「隨時資料庫」→ 加入

之後就從主畫面圖示打開,跟原生 App 一樣,選課程、拍照即上傳。

### 驗證清單

- [ ] 打開 PWA,課程按鈕有正常出現(不是「載入中」卡住)
- [ ] 拍一張照片,幾秒後「最近上傳」出現「已上傳」
- [ ] 打開 Drive 的索引表,有新的一列資料
- [ ] 打開 Drive 資料夾,照片確實在 `隨時資料庫/課程名稱/年/月/照片/` 底下

如果卡在「排隊中」不會變「已上傳」,最常見原因是 PASS_KEY 或 GAS_URL 打錯,或是部署權限沒選「任何人」— 回頭檢查 A-6。

## C. 日常使用

### 拍照

打開 App → 選課程(下面會說怎麼選/新增)→ 點「拍照上傳」→ 拍完自動排隊上傳,「最近上傳」會顯示狀態(排隊中 → 已上傳)。離線時會留在佇列裡,恢復連線後自動補傳,不用重拍。

### 課堂筆記(筆電也能用)

PWA 就是一般網頁,筆電瀏覽器打開同一個網址就能用(跟手機共用同一組連線設定跟課程清單)。「課堂筆記」卡片裡打字,存檔會自動歸到目前選的課程,存成 `.html` 檔進 Drive 對應課程的「筆記」資料夾,索引表跟「瀏覽」也看得到,跟照片、錄音一樣。存成 `.html` 是因為要保留螢光筆上色、章節標題這些格式,用瀏覽器打開會正常顯示,不是一堆標籤符號。

- **同一天、同一課程共用一個檔案**:重複按「儲存筆記」是更新同一份筆記,不會愈存愈多零碎檔案。存檔後編輯器**不會清空**——這份筆記代表「今天這堂課」,可以整堂課持續打、隨時再存一次
- 切換課程時,筆記內容會跟著換成「那個課程今天」的草稿(各自獨立,不會互相蓋掉;切回來還在)
- 打字時持續自動存草稿到瀏覽器本機(不用擔心分頁被系統關掉弄丟)
- 跟拍照一樣支援離線佇列,沒網路時先留著,恢復連線自動補傳
- 三顆小工具按鈕:
  - **⏱ 時間**:在游標處插入目前時間,例如 `[14:23]`,方便回頭對照「這句話大概是什麼時候記的」
  - **🖍 螢光筆**:選取文字後點,會**真的上黃色底色**(不是文字符號模擬);沒選取就在游標處插入一個空白標記區塊,接著打字就是螢光筆效果
  - **§ 章節**:跳出視窗問標題,插入一個章節標題,同時上方會出現一排可以點擊快速跳轉的章節導覽列——整份筆記還是同一份連續文件,不是切換分頁,但可以快速跳到任一章節,行為比較接近 Notion/Obsidian 的目錄導覽,不是 OneNote 那種真的分頁介面(取捨過:真分頁要多處理「切分頁時內容存哪、離線草稿怎麼分開存」等狀態,複雜很多,單一長文件+導覽列簡單很多,也不會有「切分頁忘記存」的風險)
- 這個功能**不是**跟 OneNote/Notability 那種「錄音+筆記時間軸同步、點筆記跳到對應錄音時間點」的功能一樣,單純是「打字筆記自動歸檔 + 螢光筆/章節導覽」;如果你要那種同步播放功能,直接用 OneNote 或 Notability(兩者的錄音+筆記同步都是免費功能),事後把匯出的筆記檔案手動丟進 Drive 對應課程資料夾,對帳工具就會自動撈進索引表
- 技術上沒有引入任何外部富文字編輯器函式庫(這個專案一路以來是單一自足的 HTML 檔案、沒有 CDN/build step),所以功能刻意精簡:沒有粗體/斜體/清單等其他格式,螢光筆只有單一黃色,沒有「取消標記」按鈕(可以用瀏覽器原生 undo,或手動選取刪除)

### 上傳錄音檔(其他裝置錄的)

如果錄音是用手機、筆電以外的其他裝置錄的(另一支手機、錄音筆、平板…),「上傳錄音檔」卡片可以直接選檔案上傳,跟拍照走同一套機制,任何裝置的瀏覽器都能用。

- **只適合短音檔**(建議 20MB 以內,例如單段對話、片段錄音)。超過 20MB 系統會跳警告,因為這條路是把整個檔案編碼後直接送給 Apps Script,而 Apps Script 對單次請求的大小有上限(約 50MB),編碼會再膨脹約 33%,檔案太大容易上傳失敗或很久
- **整堂課等級的大檔案(60~150MB)不要用這顆按鈕**,改用「分享到 Google Drive App → 存進 `_收件夾`」的方式——這個方法不限檔案大小,而且**任何裝置都能做**,不是只有手機:
  - 手機:語音備忘錄 App → 分享 → 雲端硬碟(見 [ios-shortcut-guide.md](./ios-shortcut-guide.md))
  - 筆電/其他裝置:打開 [drive.google.com](https://drive.google.com) 或裝 Google Drive 電腦版同步軟體,把錄音檔直接拖進 `隨時資料庫/_收件夾`,檔名改成課程名稱,5 分鐘內會自動被歸檔——原理完全一樣,只是換一種方式把檔案放進收件夾

### 選課程 / 新增課程

- 「目前課程」卡片裡點一個課程按鈕,選中的會反白,之後拍照都歸到這個課程
- 課程數量超過 6 個時,上面會多一排「最近使用」方便快速選,也會出現搜尋框
- 點搜尋框旁邊的「＋」新增課程,打課程名稱送出即可,新課程會直接自動選中

### 管理課程(刪除 / 瀏覽)

底部「管理課程」→ 每個課程旁邊有兩顆按鈕:

- **瀏覽** — 列出這門課目前所有照片(📷)跟錄音(🎙️),新到舊排序,點一筆直接開 Drive 檔案
- **刪除** — 只是把課程從清單移除,**不會刪除 Drive 裡已經上傳的檔案跟資料夾**。如果 Drive 那邊還留著資料夾,會出現在主畫面的「待清理提醒」卡片,提醒你要不要手動去 Drive 刪掉;如果你之後改變主意用同名字重新新增課程,提醒會自動消失

### 待清理提醒

主畫面最上面出現的黃色卡片,代表「你在 App 刪過的課程,Drive 資料夾還留著」。可以:
- 手動去 Drive 刪掉資料夾(移到垃圾桶就算,不用永久刪除)→ App 下次自動同步時提醒會自己消失
- 或是點卡片上的「已清理」手動核銷(不管 Drive 有沒有真的刪,先讓提醒消失)

### Drive 對帳(直接在 Drive 動檔案時)

系統只會自動即時索引兩條路:PWA 拍照上傳、丟進 `_收件夾` 的錄音。**如果你直接在 Drive 裡把檔案拖進/刪掉課程資料夾**,索引表跟 App「瀏覽」不會馬上反映,要靠**對帳**同步:

- 打開 App(或從其他 App 切回來)時,背景會自動對帳一次,有變更才會跳提示,平常無感,不用做任何事
- 每天凌晨 3 點也會自動對帳一次當後盾
- 想立刻同步、不想等背景觸發的話:底部「連線設定」→ 最下面「立即同步 Drive」,同時可以看到「上次同步」的時間跟結果(成功幾筆 / 失敗),藉此確認自動同步有沒有在正常運作

## D. 維護工具(需要時才用,在 Apps Script 編輯器「執行」即可,不用重新部署)

這些是手動函式,平常不用管;哪天要整理資料時,函式下拉選單選它 → 按「執行」→ 看「執行紀錄」結果。

| 函式 | 用途 |
|---|---|
| `reconcileIndex` | **對帳**:讓索引表跟 Drive 課程資料夾實際內容完全同步 — 補登你直接拖進 Drive 的檔案、清掉已刪除的列、把有檔案的課程資料夾補進課程清單、核銷待清理提醒。平常**打開 App 就會在背景自動觸發**、每日排程也會跑,這裡是需要時也能在編輯器手動跑。 |
| `cleanupDeadIndexRows` | 只清索引表裡 Drive 連結已失效的列(`reconcileIndex` 已包含這件事,這個是輕量版) |
| `syncCoursesFromFolders` | 只把 Drive 現有課程資料夾補進課程清單(不動索引表) |
| `installCleanupTrigger` | 裝上「每天凌晨 3 點自動 `reconcileIndex` 一次」的排程(只需執行一次;若之前裝過舊版清理排程,重跑一次會自動換成新的全套對帳) |

> 重點觀念:系統只會**自動即時**索引「PWA 拍照上傳」跟「丟進 `_收件夾` 的檔案」兩條路,直接動 Drive 課程資料夾要靠對帳才會反映到索引表跟 App。對帳的四種觸發方式(背景自動同步為主力)見上面「C. 日常使用 → Drive 對帳」。
